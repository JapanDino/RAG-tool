# Отчет по текущему состоянию разработки

## 1. Цель работы
Подготовить отчет о текущем состоянии проекта и зафиксировать, что уже реализовано, что требует доработки и какие ближайшие шаги нужны для перехода к полноценному прототипу.

## 2. Что реализовано на текущий момент
### 2.1. Бэкенд
- Реализован базовый API на FastAPI со слоями для датасетов, документов, чанков и аннотаций Блума.
- Добавлен базовый маршрут анализа текста `/analyze`, который:
  - разбивает входной текст на предложения (простое семантическое приближение),
  - формирует вероятности по уровням Блума на основе ключевых глаголов.

### 2.2. Инфраструктура
- Поддерживается асинхронная обработка задач через Celery.
- Есть базовый экспорт данных и поиск по эмбеддингам.

## 3. Оценка качества текущего состояния
### 3.1. Сильные стороны
- Есть сквозная инфраструктурная база (API, БД, задачи, экспорт).
- Сформирована структура хранения для узлов и аннотаций.

### 3.2. Ограничения
- Алгоритм выделения узлов пока упрощен: разбиение по предложениям не отражает реальную семантику.
- Классификация по Блуму реализована через ключевые слова, а не через ML/LLM.
- Визуализация графа знаний пока отсутствует.

## 4. Рекомендации и следующие шаги
1. **Заменить эвристики** на полноценный NER/Chunking и классификатор с вероятностями.
2. **Добавить вычисление связей** между узлами (семантическая близость, ко‑встречаемость).
3. **Реализовать UI вкладки** «Анализ контента» и «Граф знаний».
4. **Подготовить тестовый датасет** с экспертной разметкой для оценки качества.

## 5. Вывод
Сформирована техническая основа проекта. Текущая реализация пригодна для демонстрации базового потока обработки текста, но требует замены эвристик на полноценные модели и реализации визуализации графа, чтобы соответствовать требованиям ТЗ.
